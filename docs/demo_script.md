# 演示视频脚本

## 目标时长

2 到 3 分钟。

## 开场

大家好，这是我做的多 Agent 自动化办公工作流平台。它面向企业办公场景，用户上传销售 Excel 并输入任务目标后，系统会自动拆解任务，由多个 Agent 协作完成数据分析、报告生成和质量审核。

## 场景说明

这里我选择的是销售数据分析场景。传统做法需要人工打开 Excel、统计区域销售额、找出热销品类，再写成经营报告。这个项目把这条流程自动化了。

## 操作流程

1. 上传示例销售数据。
2. 输入任务目标：分析销售数据并生成经营报告。
3. 系统创建任务，Planner Agent 先拆解执行计划。
4. 我在人工确认节点检查计划，确认后继续执行。
5. Data Analyst Agent 调用数据分析工具，统计销售额、区域排行和品类排行。
6. Writer Agent 生成 Markdown 报告。
7. Reviewer Agent 审核报告是否包含关键指标、结论和建议。
8. 最终页面展示报告和完整执行 Trace。

## 技术讲解

后端使用 FastAPI，任务和执行步骤保存在 SQLite。工作流层按 Planner、Data Analyst、Writer、Reviewer 拆分职责，每个节点都有输入、输出、状态和错误记录。第一版使用本地规则保证无 Key 也能演示，后续可以把节点替换成 LangGraph 的 `StateGraph` 和真实大模型调用。

## 结束语

这个项目重点不是简单调用一次大模型，而是展示一个企业数字员工系统需要的任务编排、工具调用、多 Agent 协作、人工确认、失败恢复和 Trace 可观测能力。
