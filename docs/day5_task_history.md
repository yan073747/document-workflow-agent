# 第 5 天：任务历史列表和详情复盘

## 今日目标

让系统从“单次任务 Demo”升级成“可复盘的任务工作台”。

完成后，用户不仅能创建和执行一个任务，还能查看过去的任务，点击某个任务恢复它的计划、Trace、报告和下载按钮。

## 已完成内容

- `GET /api/tasks` 改为返回任务摘要列表。
- 任务摘要包含：
  - 任务 ID
  - 任务目标
  - 当前状态
  - 是否已生成报告
  - 计划步骤数
  - Trace 数量
  - 创建时间
  - 更新时间
- 前端新增“任务历史”面板。
- 支持历史任务状态筛选：
  - 全部
  - 等待确认
  - 已完成
  - 失败
- 点击历史任务后，自动加载完整详情。
- 详情复盘会恢复：
  - 执行计划
  - Agent Trace
  - Markdown 报告
  - Markdown/PDF 下载按钮
- 当前选中的历史任务会高亮显示。

## 接口说明

### 任务摘要列表

```text
GET /api/tasks
```

返回示例：

```json
{
  "items": [
    {
      "id": "task-id",
      "objective": "分析销售数据并生成经营报告。",
      "status": "completed",
      "has_report": true,
      "plan_step_count": 3,
      "trace_count": 9,
      "created_at": "2026-07-11T05:22:26",
      "updated_at": "2026-07-11T05:22:26"
    }
  ]
}
```

### 任务详情

```text
GET /api/tasks/{task_id}
```

返回完整任务对象和 Trace，用于前端复盘。

## 为什么这一步重要

真实的办公自动化平台不能只处理一次任务。用户需要知道：

- 以前跑过哪些任务。
- 哪些任务完成了。
- 哪些任务失败了。
- 每个任务是怎么执行的。
- 报告是否还能重新下载。

任务历史让项目更接近企业工作流平台，也更方便面试演示。
